The Problem

  • 🧗‍♂️Knowledge Silos: Developers write code, but hate writing documentation.
  • ⏳Outdated Docs: Code changes rapidly, docs stay the same.
  • 🤯Onboarding Hell: New developers spend weeks understanding undocumented spaghetti code.

The AutoDoc Solution

An event-driven, language-agnostic pipeline that automates technical documentation within the CI/CD lifecycle.

  • Push Code to GitHub.
  • Webhook triggers the pipeline automatically.
  • AI reads the diffs and updates autodoc.md.
  • Bot opens a Pull Request magically.

4-Tier Architecture

1. Webhook Service (Node.js) Listens to GitHub, extracts diffs/tree.
Message Broker (RabbitMQ) Decouples services, handles async queues.
2 & 3. AI Parsing & Docs (Python) Prepares context & calls Gemini LLM.
4. PR Service (Node.js) Commits to GitHub & opens PRs.

The "Cold Start" Engine

Innovation

What happens if autodoc.md doesn't exist yet?
Sending a single commit diff to AI yields terrible results. Our solution:

  • Init Mode: If 404, we fetch the entire Git Tree.
  • Smart Filtering: Ignores node_modules, .git, etc.
  • Full Context: Feeds up to 30 core files to AI to map out the entire project architecture from scratch.

Infinite Loop Prevention

Merging our own autodoc.md PR creates a new Push event, triggering the system again!
